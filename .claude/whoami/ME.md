# Me

I'm a data engineer at MetLife Korea, a life insurance company. My day-to-day work involves operating on-premises ETL 
pipelines built on Informatica PowerCenter, while also researching and advocating for modern data engineering practices 
within the organization.

Recently I co-led the initialization of a Databricks platform on Azure — our next-generation data warehousing, BI, and 
AI infrastructure. Alongside that, I maintain Azure Data Factory pipelines that ingest on-premises data into Azure 
(ADLS/Blob), making it available to downstream Databricks workloads. I also own the pseudonymization layer over 
sensitive data, ensuring that downstream usage complies with Korea's Personal Information Protection Act (개인정보보호법).

On the technical side, my interests lie in open table formats (Delta Lake, Iceberg — haven't touched Hudi yet), 
in-memory columnar processing with Arrow and Polars, and writing clean Python with correct use of concurrency and 
threading. I'm a fan of uv for Python environment management. Beyond that, I follow developments in query engines like 
Trino, metadata management platforms like OpenMetadata, and data lineage tooling such as OpenLineage.
